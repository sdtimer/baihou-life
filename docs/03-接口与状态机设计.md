## 1. 通用约定

### 1.1 基础信息

| 项目 | 说明 |
|:---|:---|
| Base URL | `https://api.baihou.life`（生产）/ `http://localhost:8080`（开发） |
| 协议 | HTTPS（生产强制） |
| 编码 | UTF-8 |
| Content-Type | `application/json` |
| 版本策略 | URL Path 版本化（`/v1/`、`/v2/`） |

### 1.2 鉴权方式

基于若依框架的 Token 鉴权机制。小程序端使用自定义 JWT，CMS 后台复用若依原有的 Token 机制。

所有需要认证的接口在 HTTP Header 中携带：
```
Authorization: Bearer <jwt_token>
```

| 鉴权等级 | 标记 | 说明 |
|:---|:---|:---|
| 公开 | `PUBLIC` | 无需 Token |
| 登录用户 | `USER` | 需有效 Token，role >= 1 |
| 设计师 | `DESIGNER` | 需有效 Token，role == 2 |
| 管理员 | `ADMIN` | 需有效 Token，role == 99（若依后台管理员） |

### 1.3 统一响应格式

与若依风格保持一致，使用 `AjaxResult` 封装：

**成功响应**：
```json
{
  "code": 200,
  "msg": "操作成功",
  "data": { ... }
}
```

**分页响应**（若依 `TableDataInfo` 风格）：
```json
{
  "code": 200,
  "msg": "查询成功",
  "rows": [ ... ],
  "total": 156
}
```

**错误响应**：
```json
{
  "code": 403,
  "msg": "权限不足：仅设计师可下载素材"
}
```

### 1.4 全局错误码

| 错误码 | 含义 | 触发场景 |
|:---|:---|:---|
| 200 | 成功 | 正常响应 |
| 400 | 参数错误 | 缺少必填字段、格式不合法 |
| 401 | 未认证 | Token 缺失或已过期 |
| 403 | 权限不足 | 角色不满足接口要求 |
| 404 | 资源不存在 | ID 不存在或已被删除 |
| 409 | 状态冲突 | 重复操作（如重复留资、已关闭订单重复支付） |
| 422 | 业务校验失败 | 区域不支持、商品已下架、库存不足 |
| 429 | 频率限制 | 同一用户/IP 超出请求频率 |
| 500 | 服务器错误 | 未预期异常 |

### 1.5 通用请求参数

| 参数 | 位置 | 说明 |
|:---|:---|:---|
| `Authorization` | Header | JWT Token（需鉴权接口必传） |
| `X-Region-Id` | Header | 当前选择的区域 ID（全局携带） |

### 1.6 若依框架集成说明

| 方面 | 说明 |
|:---|:---|
| Controller 层 | 小程序端接口放在 `com.ruoyi.baihou.controller.api` 包下，CMS 接口放在 `com.ruoyi.baihou.controller.admin` 包下 |
| Service 层 | 业务逻辑在 `com.ruoyi.baihou.service` 包下实现 |
| Mapper 层 | MyBatis Mapper 在 `com.ruoyi.baihou.mapper` 包下，XML 在 `resources/mapper/baihou/` 下 |
| 鉴权 | 小程序端自定义 `WxTokenFilter`，CMS 端复用若依 `TokenService` |
| 分页 | 复用若依 `startPage()` + `getDataTable()` 分页封装 |

### A01 微信登录

**POST** `/v1/auth/wechat-login`　　**鉴权**：`PUBLIC`

**请求体**：

| 参数 | 类型 | 必填 | 说明 |
|:---|:---|:---:|:---|
| `code` | String | ✓ | `wx.login()` 获取的临时 code |
| `encrypted_data` | String | ✗ | 加密手机号数据 |
| `iv` | String | ✗ | 加密向量 |

**响应体**：
```json
{
  "code": 200,
  "data": {
    "token": "eyJhbGciOiJIUzI1NiIs...",
    "expires_in": 604800,
    "user": {
      "uid": 100001,
      "role": 1,
      "phone": "138****1234",
      "nickname": "微信用户",
      "avatar_url": "https://cdn.baihou.life/avatar/default.png",
      "region_id": "foshan"
    }
  }
}
```

**业务规则**：
- 后端调用微信 `code2Session` 接口换取 `openid`
- 若 `openid` 匹配已有设计师预创建记录（手机号匹配），`role` 返回 2
- 新用户自动创建，`role` 默认为 1
- Token 有效期 7 天，存入 Redis（复用若依 Token 缓存机制）

### A03 商品列表

**GET** `/v2/product/list`　　**鉴权**：`PUBLIC`（价格字段按角色动态返回）

**请求参数**（Query）：

| 参数 | 类型 | 必填 | 说明 |
|:---|:---|:---:|:---|
| `region_id` | String | ✓ | 区域 ID |
| `category_id` | Long | ✗ | 品类 ID |
| `space` | String | ✗ | 空间标签（如 `kitchen_bath`） |
| `scene` | String | ✗ | 场景标签（如 `minimalist`） |
| `keyword` | String | ✗ | 搜索关键词（匹配名称/品牌） |
| `sort_by` | String | ✗ | 排序：`default` / `price_asc` / `price_desc` / `newest` |
| `pageNum` | Int | ✗ | 页码（默认 1，若依风格） |
| `pageSize` | Int | ✗ | 每页数量（默认 20，最大 50） |

**响应体**：
```json
{
  "code": 200,
  "rows": [
    {
      "id": 10001,
      "name": "TOTO 浴室柜 800mm",
      "sku_code": "TOTO-YSG-800",
      "brand": "TOTO",
      "category_id": 3,
      "category_name": "洁具",
      "guide_price": 1280.00,
      "designer_price": 1088.00,
      "price_unit": "元/套",
      "cover_image": "https://cdn.baihou.life/product/10001/cover_400.webp",
      "space_tags": ["kitchen_bath"],
      "scene_tags": ["minimalist", "healing"]
    }
  ],
  "total": 56
}
```

**业务规则**：
- `region_id` 必传，未传返回 400
- `guide_price`：游客返回 `null`，客户/设计师返回实际值
- `designer_price`：仅 `role==2` 时返回，其他角色不含此字段
- 三个筛选条件（category_id / space / scene）为 AND 逻辑
- 后端使用 MySQL JSON 函数实现区域和标签过滤

**MyBatis SQL 要点**：
```xml
<where>
  p.status = 'on_shelf' AND p.del_flag = '0'
  AND (JSON_CONTAINS(p.regions, '"ALL"') OR JSON_CONTAINS(p.regions, CONCAT('"', #{regionId}, '"')))
  <if test="categoryId != null">AND p.category_id = #{categoryId}</if>
  <if test="space != null">AND JSON_CONTAINS(p.space_tags, CONCAT('"', #{space}, '"'))</if>
  <if test="scene != null">AND JSON_CONTAINS(p.scene_tags, CONCAT('"', #{scene}, '"'))</if>
  <if test="keyword != null">AND (p.name LIKE CONCAT('%', #{keyword}, '%') OR p.brand LIKE CONCAT('%', #{keyword}, '%'))</if>
</where>
```

### A10 创建订单

**POST** `/v1/order/create`　　**鉴权**：`USER`　　**优先级**：P1

**请求体**：

| 参数 | 类型 | 必填 | 说明 |
|:---|:---|:---:|:---|
| `region_id` | String | ✓ | 下单区域 |
| `items` | Array | ✓ | 订单项列表 |
| `items[].product_id` | Long | ✓ | 商品 ID |
| `items[].quantity` | Int | ✓ | 数量（>=1） |
| `remark` | String | ✗ | 备注 |

**业务规则**：
- 按用户角色计算价格（客户=指导价，设计师=折扣价），价格快照到 OrderItem
- 订单创建后 `pending_pay`，30 分钟未支付自动关闭（若依 Quartz 定时任务）
- `order_no` 格式：`ORD` + `yyyyMMdd` + 8 位序号
- 校验 `product_id` 在区域范围内且上架

### A13 发起微信支付

**POST** `/v1/payment/wechat/prepay`　　**鉴权**：`USER`　　**优先级**：P1

**请求体**：`{ "order_no": "ORD2026030200000001" }`

**响应体**：返回微信支付参数（appId / timeStamp / nonceStr / package / signType / paySign），前端直接传入 `wx.requestPayment()`。

**业务规则**：
- 校验订单属于当前用户且状态为 `pending_pay`
- 调用微信统一下单 API（V3），签名使用商户证书私钥

---

### A14 微信支付回调

**POST** `/v1/payment/notify`　　**鉴权**：`PUBLIC`（微信签名验证）

**处理逻辑**：
1. 验证微信签名
2. 解析报文，提取 order_no 和 transaction_id
3. 幂等判断（订单已 paid 则直接返回成功）
4. 更新订单状态 `pending_pay → paid`
5. 写入 `transaction_id` 和 `paid_at`
6. 返回成功应答给微信

**约束**：5 秒内返回，需处理重复通知。

## 4. 状态机设计

### 状态机通用规则

每个状态机定义：
- **状态枚举**：所有可能的状态值（Java 中建议用枚举类定义在 `com.ruoyi.baihou.enums` 包下）
- **转换规则**：允许从哪个状态转到哪个状态
- **触发条件**：什么操作/事件触发转换
- **操作方**：谁有权触发（前台/后台/系统定时任务）

后端状态流转统一通过 Service 层校验，禁止直接操作数据库改状态。

---

### S1 用户状态机

**状态枚举**：

| 状态值 | 名称 | 说明 |
|:---|:---|:---|
| `anonymous` | 游客 | 未登录状态 |
| `active_customer` | 已激活客户 | 已登录普通客户 |
| `active_designer` | 已激活设计师 | 已授权设计师 |
| `disabled` | 已停用 | 账号被管理员停用 |

**状态转换图**：

```
                 ┌──────────────────────────┐
                 │                          │
                 ▼                          │
┌──────────┐  登录成功  ┌──────────────────┐ │
│ anonymous │ ───────▶ │ active_customer   │ │
│  (游客)    │          │  (已激活客户)       │ │
└──────────┘          └───────┬──────────┘ │
                              │             │ Token过期/主动退出
                     后台关联设计师记录         │
                              │             │
                              ▼             │
                     ┌──────────────────┐   │
                     │ active_designer   │ ──┘
                     │  (已激活设计师)      │
                     └───────┬──────────┘
                              │
                      管理员停用 │ 管理员启用
                              ▼
                     ┌──────────────────┐
                     │    disabled       │
                     │   (已停用)         │
                     └──────────────────┘
```

**转换规则表**：

| 当前状态 | 目标状态 | 触发条件 | 操作方 |
|:---|:---|:---|:---|
| `anonymous` | `active_customer` | 微信登录成功 | 系统 |
| `anonymous` | `active_designer` | 登录且手机号匹配设计师记录 | 系统 |
| `active_customer` | `active_designer` | 管理员创建设计师并关联手机号 | 后台 |
| `active_customer` | `anonymous` | Token 过期 / 主动退出 | 前台/系统 |
| `active_designer` | `anonymous` | Token 过期 / 主动退出 | 前台/系统 |
| `active_customer` | `disabled` | 管理员停用 | 后台 |
| `active_designer` | `disabled` | 管理员停用 | 后台 |
| `disabled` | `active_customer` | 管理员重新启用 | 后台 |
| `disabled` | `active_designer` | 管理员重新启用（有设计师记录） | 后台 |

### S2 商品状态机

**状态枚举**：

| 状态值 | 名称 | 说明 |
|:---|:---|:---|
| `draft` | 草稿 | 新建未上架，前台不可见 |
| `on_shelf` | 已上架 | 前台可见可购买 |
| `off_shelf` | 已下架 | 前台不可见 |
| `archived` | 已归档 | 软删除，仅后台可见（终态） |

**状态转换图**：

```
┌────────┐  发布上架   ┌──────────┐
│  draft  │ ─────────▶│ on_shelf  │
│ (草稿)  │           │ (已上架)   │
└────┬───┘           └──┬────┬──┘
     │                  │    │
     │    ┌─────────────┘    │
     │    │ 下架              │ 归档
     │    ▼                  │
     │  ┌──────────┐         │
     │  │ off_shelf │         │
     │  │ (已下架)   │         │
     │  └──┬────┬──┘         │
     │     │    │ 重新上架      │
     │     │    └─────────────┤
     │     │ 归档              │
     │     ▼                  ▼
     │  ┌──────────────────────┐
     └─▶│      archived        │
        │      (已归档)          │
        └──────────────────────┘
```

**转换规则表**：

| 当前状态 | 目标状态 | 触发条件 | 操作方 |
|:---|:---|:---|:---|
| `draft` | `on_shelf` | 管理员"发布上架" | 后台 |
| `draft` | `archived` | 管理员删除草稿 | 后台 |
| `on_shelf` | `off_shelf` | 管理员手动下架 | 后台 |
| `on_shelf` | `off_shelf` | 关联品类被停用 | 系统 |
| `on_shelf` | `archived` | 管理员直接归档 | 后台 |
| `off_shelf` | `on_shelf` | 管理员重新上架 | 后台 |
| `off_shelf` | `archived` | 管理员归档 | 后台 |

**上架前置条件**：商品必须有 name、sku_code、category_id、至少一个 region、至少一张图片。

### S3 预约状态机

**状态枚举**：

| 状态值 | 名称 | 说明 |
|:---|:---|:---|
| `pending` | 待确认 | 客户提交，等待后台确认 |
| `confirmed` | 已确认 | 后台已确认 |
| `in_progress` | 进行中 | 服务人员已上门 |
| `completed` | 已完成 | 服务完成（终态） |
| `cancelled` | 已取消 | 预约取消（终态） |

**状态转换图**：

```
┌──────────┐  后台确认   ┌───────────┐  开始服务  ┌─────────────┐  完成  ┌───────────┐
│  pending  │ ────────▶ │ confirmed  │ ───────▶ │ in_progress  │ ────▶ │ completed  │
│ (待确认)  │           │ (已确认)    │          │ (进行中)      │       │ (已完成)    │
└──┬───────┘           └──┬────────┘          └─────────────┘       └───────────┘
   │                      │
   │ 客户取消/后台拒绝        │ 客户取消(24h内)/后台取消
   ▼                      ▼
┌──────────────────────────────┐
│          cancelled            │
│          (已取消)               │
└──────────────────────────────┘
```

**转换规则表**：

| 当前状态 | 目标状态 | 触发条件 | 操作方 |
|:---|:---|:---|:---|
| `pending` | `confirmed` | 后台确认预约 | 后台 |
| `pending` | `cancelled` | 客户取消 / 后台拒绝 | 前台/后台 |
| `confirmed` | `in_progress` | 服务人员开始服务 | 后台 |
| `confirmed` | `cancelled` | 客户取消（确认后 24h 内）/ 后台取消 | 前台/后台 |
| `in_progress` | `completed` | 服务完成 | 后台 |

**约束**：`in_progress` 不可取消。确认后超 24 小时客户取消需电话联系客服。

### S4 订单状态机

**状态枚举**：

| 状态值 | 名称 | 说明 |
|:---|:---|:---|
| `pending_pay` | 待支付 | 订单已创建，等待支付 |
| `paid` | 已支付 | 支付成功 |
| `processing` | 处理中 | 商家确认处理 |
| `shipped` | 已发货 | 商品已发出 |
| `completed` | 已完成 | 确认收货（终态） |
| `closed` | 已关闭 | 超时/取消/异常（终态） |

**状态转换图**：

```
┌──────────────┐  支付成功   ┌────────┐  后台确认  ┌────────────┐  发货   ┌──────────┐
│ pending_pay   │ ─────────▶│  paid   │ ───────▶ │ processing  │ ─────▶│ shipped   │
│ (待支付)       │           │(已支付) │          │ (处理中)     │       │ (已发货)   │
└──┬───────────┘           └──┬─────┘          └──┬─────────┘       └──┬───────┘
   │                          │                    │                    │
   │ 超时(30min)/              │ 后台关闭            │ 后台关闭             │ 确认收货/
   │ 客户取消                   │ (异常退款)           │ (异常退款)            │ 自动15天
   ▼                          ▼                    ▼                    ▼
┌─────────────────────────────────────┐         ┌──────────────────────────┐
│              closed                  │         │       completed           │
│             (已关闭)                  │         │       (已完成)              │
└─────────────────────────────────────┘         └──────────────────────────┘
```

**转换规则表**：

| 当前状态 | 目标状态 | 触发条件 | 操作方 |
|:---|:---|:---|:---|
| `pending_pay` | `paid` | 微信支付回调确认 | 系统 |
| `pending_pay` | `closed` | 超过 30 分钟未支付 | 系统（Quartz 定时任务） |
| `pending_pay` | `closed` | 客户主动取消 | 前台 |
| `paid` | `processing` | 后台确认订单 | 后台 |
| `paid` | `closed` | 后台关闭（需退款，一期线下处理） | 后台 |
| `processing` | `shipped` | 后台确认发货（填物流单号） | 后台 |
| `processing` | `closed` | 后台关闭（需退款） | 后台 |
| `shipped` | `completed` | 客户确认收货 | 前台 |
| `shipped` | `completed` | 发货 15 天自动确认 | 系统（Quartz） |

**定时任务配置（ruoyi-quartz）**：
- `OrderTimeoutJob`：每分钟扫描 `status='pending_pay'` 且 `expires_at < NOW()` 的订单，自动关闭
- `OrderAutoCompleteJob`：每天凌晨扫描 `status='shipped'` 且 `shipped_at < NOW() - 15天` 的订单，自动完成

### S5 线索状态机

**状态枚举**：

| 状态值 | 名称 | 说明 |
|:---|:---|:---|
| `new` | 未跟进 | 新建线索 |
| `following` | 跟进中 | 已分配，正在跟进 |
| `converted` | 已成单 | 线索转化（终态） |
| `abandoned` | 已放弃 | 线索无效 |

**状态转换图**：

```
┌──────────┐  分配/开始跟进  ┌──────────┐  成单   ┌──────────┐
│   new     │ ────────────▶│ following │ ─────▶ │ converted │
│ (未跟进)  │              │ (跟进中)  │        │ (已成单)   │
└──┬───────┘              └──┬────┬──┘        └──────────┘
   │                         │    │
   │ 判定无效                  │    │ 无意向/无效
   ▼                         │    ▼
┌──────────┐                 │  ┌──────────┐
│ abandoned │ ◀──────────────┘  │ abandoned │
│ (已放弃)   │   重新激活         │ (已放弃)   │
└──────────┘ ◀──────────────── └──────────┘
```

**转换规则表**：

| 当前状态 | 目标状态 | 触发条件 | 操作方 |
|:---|:---|:---|:---|
| `new` | `following` | 分配销售 / 开始跟进 | 后台/自动分配 |
| `new` | `abandoned` | 判定无效 | 后台 |
| `following` | `converted` | 客户下单/预约成功 | 后台 |
| `following` | `abandoned` | 客户明确放弃 / 无法联系 | 后台 |
| `abandoned` | `following` | 重新激活跟进 | 后台 |

**约束**：`converted` 为终态。`abandoned` 可回退到 `following`。48 小时未分配系统提醒，跟进中 7 天无操作系统标记提醒。
