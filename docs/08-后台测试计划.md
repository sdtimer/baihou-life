## 2. 测试目标

### 2.1 总体目标

- 验证后台 `B1-B8` 八个模块在一期范围内可稳定使用
- 验证后台接口、状态机、权限、导出、定时任务等核心能力符合既有设计
- 验证商品管理与 Banner 管理最新改造对原系统无破坏性影响
- 验证公共主数据能力（品类、区域、字典、规格模板）可支撑后台各模块联动
- 为预发验收与上线准备提供可执行的测试依据与结果口径

### 2.2 本次测试主线

本轮测试以**后台变更回归**为主线，重点覆盖以下改造：

- 商品管理模块改造
  - 区域字段由手填 JSON 改为多选下拉
  - 空间标签、场景标签改为数据字典多选
  - 规格参数改为“品类规格模板 + 动态表单”
- Banner 管理模块改造
  - 区域字段改为多选下拉
  - 引入 `ALL` 语义与互斥逻辑
  - 表格列改为可读名称 Tag 展示
- 区域公共能力改造
  - 新增 `GET /admin/regions/options`
  - 商品与 Banner 共同复用区域选项接口
- 本地联调环境基线
  - 按 `service_commands.md` 验证后端与管理端启动、连通与依赖完整性

## 4. 测试依据与关键约束

### 4.1 接口约束

后台接口统一遵循 `/admin/` 路由规范，鉴权等级均为 `ADMIN`。本次测试需重点覆盖以下接口族：

- `/admin/products`
- `/admin/categories`
- `/admin/regions`
- `/admin/designers`
- `/admin/designer-groups`
- `/admin/leads`
- `/admin/banners`
- `/admin/appointments`
- `/admin/orders`

同时新增或追加验证以下改造接口：

- `GET /admin/regions/options`
- `GET /admin/categories/{categoryId}/spec-defs`
- `POST /admin/categories/{categoryId}/spec-defs`
- `PUT /admin/categories/{categoryId}/spec-defs/{id}`
- `DELETE /admin/categories/{categoryId}/spec-defs/{id}`
- `PUT /admin/categories/{categoryId}/spec-defs/sort`

### 4.2 状态机约束

所有状态流转统一通过 Service 层校验，禁止直接改库。测试必须验证合法流转、非法流转、终态拦截、联动规则与定时任务影响。

| 对象 | 状态值 |
| :--- | :--- |
| 商品 | `draft / on_shelf / off_shelf / archived` |
| 预约 | `pending / confirmed / in_progress / completed / cancelled` |
| 订单 | `pending_pay / paid / processing / shipped / completed / closed` |
| 线索 | `new / following / converted / abandoned` |

### 4.3 改造约束

#### 商品管理改造约束

- `regions` 继续存 JSON 数组字符串，前端仅变更为多选下拉录入
- `space_tags`、`scene_tags` 继续存 JSON 数组字符串，但选项改由数据字典提供
- `spec_params` 继续存 JSON 对象，字段结构受品类规格模板约束
- 商品上架仍需满足原前置条件：名称、SKU、品类、至少一个区域、至少一张图片

#### Banner 管理改造约束

- `regions` 存储格式不变，仍为 JSON 数组字符串
- `[`ALL`]` 表示全部区域
- `ALL` 与具体区域互斥
- 手动勾选全部具体区域时，不自动转义为 `ALL`

#### 区域 options 公共能力约束

- 仅返回 `is_active = 1` 的区域
- 返回顺序按 `sort_order` 升序
- 商品与 Banner 共用该接口，测试需验证一致性与稳定性

```bash
mvn -s '../maven-central-settings.xml' -Dmaven.repo.local='../.m2/repository' -pl ruoyi-admin spring-boot:run
```

- API 地址可访问：`http://localhost:8080`

#### 管理端启动

在 `backend/ruoyi-baihou-life/ruoyi-ui-vue3/` 目录执行：

```bash
npm run dev
```

验证口径：

- 浏览器可访问：`http://localhost:80`
- 页面可正常登录并进入后台首页

## 9. 风险与应对

| 风险 | 说明 | 应对措施 |
| :--- | :--- | :--- |
| 区域 options 接口被多模块复用 | 一处异常会影响商品与 Banner 两个模块 | 将 `GET /admin/regions/options` 列为 P0 公共回归项 |
| 字典键值与历史数据不一致 | 可能导致标签无法正确回填或展示 | 单列存量数据兼容测试，并确认是否需要数据迁移 |
| 规格模板引入后链路变复杂 | 商品新增、编辑、上架校验耦合品类模板 | 加强 B1/B2 联动回归与异常场景覆盖 |
| 历史 Banner 数据包含停用区域或未知 ID | 下拉不显示但历史数据仍存在 | 增加展示兼容、编辑回填、保存行为验证 |
| 环境未准备完整 | MySQL / Redis / 字典 / 新表缺失导致无法联调 | 测试前执行环境检查清单 |
| 权限配置不一致 | 菜单、按钮、接口权限可能不一致 | 逐模块校验菜单、按钮、接口三个层级 |

| draft | 上架（缺区域或缺图片） | 拒绝，提示校验失败 |
| draft | 归档 | 转 `archived` |
| on_shelf | 下架 | 转 `off_shelf` |
| on_shelf | 归档 | 转 `archived` |
| off_shelf | 重新上架 | 转 `on_shelf` |
| archived | 任意可操作状态 | 拒绝 |
