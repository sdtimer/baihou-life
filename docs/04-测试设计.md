## 1. 测试策略

### 1.1 测试范围

| 测试层次 | 覆盖范围 | 工具/方式 |
|:---|:---|:---|
| 单元测试 | 后端 Service 层核心业务逻辑 | JUnit 5 + Mockito（若依已集成） |
| 接口测试 | 全部 API 接口（A01-A14、B01-B08） | Postman / Apifox + 自动化脚本 |
| 集成测试 | 前后端联调、微信登录流程、支付流程 | 手工测试 + 微信开发者工具 |
| UI 测试 | 小程序各页面交互、兼容性 | 真机测试（iOS + Android） |
| 性能测试 | 商品列表首屏响应 < 1s | JMeter 压测 |
| 安全测试 | 越权访问、素材防盗链、数据加密 | 手工渗透 + 自动化扫描 |

### 1.2 测试优先级

| 优先级 | 模块 | 说明 |
|:---|:---|:---|
| P0 | M1 身份认证、M3 选品中心、M4 商品详情 | 核心流程，必须通过 |
| P0 | M2 区域化、M6 留资、M5 素材下载 | 核心业务，必须通过 |
| P1 | M7 预约、M8 订单支付 | 重要功能，一期需覆盖 |
| P1 | B1-B8 CMS 后台 | 后台管理功能 |

### 1.3 测试环境

| 环境 | 说明 |
|:---|:---|
| 开发环境 | 本地 / 开发服务器，MySQL + Redis，微信测试号 |
| 测试环境 | 独立服务器，模拟生产配置，微信沙箱支付 |
| 预生产 | 与生产同配置，最终验收测试 |
