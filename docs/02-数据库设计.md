## 2. 数据模型总览

### 2.1 业务表清单

| 表名 | 模块 | 用途 |
| :--- | :--- | :--- |
| `bh_region` | B3 区域管理 | 区域主数据 |
| `bh_category` | B2 品类管理 | 品类树主数据 |
| `bh_category_spec_def` | B2 / B1 | 品类规格模板定义 |
| `bh_product` | B1 商品管理 | 商品主表 |
| `bh_media` | B1 商品管理 | 商品媒体资源 |
| `bh_banner` | B6 Banner 管理 | 首页 Banner / 推荐位 |
| `bh_wx_user` | B4 设计师管理 / 用户域 | 微信用户主表 |
| `bh_designer_group` | B4 设计师管理 | 设计师分组 |
| `bh_designer` | B4 设计师管理 | 设计师账号 |
| `bh_lead` | B5 线索管理 | 留资线索 |
| `bh_appointment` | B7 预约管理 | 服务预约 |
| `bh_order` | B8 订单管理 | 订单主表 |

### 2.2 关系说明

- `bh_category` 与 `bh_product` 为一对多
- `bh_category` 与 `bh_category_spec_def` 为一对多
- `bh_product` 与 `bh_media` 为一对多
- `bh_designer_group` 与 `bh_designer` 为一对多
- `bh_wx_user` 可与 `bh_designer` 通过 `user_id` 关联
- `bh_region` 与 `bh_product` / `bh_banner` / `bh_lead` / `bh_appointment` / `bh_order` 通过 `region_id` 或 JSON 区域数组关联

### 2.3 当前实现特征

- 商品与 Banner 的区域字段均使用 JSON 存储
- 商品的空间标签、场景标签使用 JSON 数组存储，但选项来源于若依字典
- 商品规格参数使用 JSON 对象存储，结构由 `bh_category_spec_def` 约束
- 当前 SQL 脚本未提供 `bh_order_item`、`bh_download_log` 的落地建表脚本，因此本版文档不再将其列为已实现业务表
