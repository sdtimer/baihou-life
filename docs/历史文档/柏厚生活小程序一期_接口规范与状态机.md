# 第一部分 接口规范

## 通用约定

### 基础信息

| 项目 | 说明 |
|:---|:---|
| Base URL | `https://api.baihou.life` |
| 协议 | HTTPS（强制） |
| 编码 | UTF-8 |
| Content-Type | `application/json` |
| 版本策略 | URL Path 版本化（/v1/、/v2/） |

### 鉴权方式

所有需要认证的接口在 HTTP Header 中携带：

```
Authorization: Bearer <jwt_token>
```

| 鉴权等级 | 标记 | 说明 |
|:---|:---|:---|
| 公开 | `PUBLIC` | 无需 Token |
| 登录用户 | `USER` | 需有效 Token，role >= 1 |
| 设计师 | `DESIGNER` | 需有效 Token，role == 2 |
| 管理员 | `ADMIN` | 需有效 Token，role == 99 |

### 统一响应格式

**成功响应**：

```json
{
  "code": 200,
  "message": "success",
  "data": { ... },
  "timestamp": 1709366400000
}
```

**分页响应**：

```json
{
  "code": 200,
  "message": "success",
  "data": {
    "list": [ ... ],
    "pagination": {
      "page": 1,
      "page_size": 20,
      "total": 156,
      "total_pages": 8
    }
  },
  "timestamp": 1709366400000
}
```

**错误响应**：

```json
{
  "code": 403,
  "message": "权限不足：仅设计师可下载素材",
  "data": null,
  "timestamp": 1709366400000
}
```

### 全局错误码

| 错误码 | 含义 | 触发场景 |
|:---|:---|:---|
| 200 | 成功 | 正常响应 |
| 400 | 参数错误 | 缺少必填字段、格式不合法 |
| 401 | 未认证 | Token 缺失或已过期 |
| 403 | 权限不足 | 角色不满足接口要求 |
| 404 | 资源不存在 | ID 不存在或已被删除 |
| 409 | 状态冲突 | 重复操作（如重复留资、已关闭订单重复支付） |
| 422 | 业务校验失败 | 区域不支持、商品已下架、库存不足 |
| 429 | 频率限制 | 同一用户/IP 超出请求频率 |
| 500 | 服务器错误 | 未预期异常 |

### 通用请求参数

| 参数 | 位置 | 说明 |
|:---|:---|:---|
| `Authorization` | Header | JWT Token（需鉴权接口必传） |
| `X-Region-Id` | Header | 当前选择的区域 ID（全局携带，可选，部分接口也支持 Query 传参） |

### A01 微信登录

**POST** `/v1/auth/wechat-login`

**鉴权**：`PUBLIC`

**请求体**：

| 参数 | 类型 | 必填 | 说明 |
|:---|:---|:---:|:---|
| `code` | String | ✓ | `wx.login()` 获取的临时 code |
| `encrypted_data` | String | ✗ | 加密手机号数据（`getPhoneNumber` 回调） |
| `iv` | String | ✗ | 加密向量 |

**响应体**：

```json
{
  "code": 200,
  "data": {
    "token": "eyJhbGciOiJIUzI1NiIs...",
    "expires_in": 604800,
    "user": {
      "uid": 100001,
      "role": 1,
      "phone": "138****1234",
      "nickname": "微信用户",
      "avatar_url": "https://cdn.baihou.life/avatar/default.png",
      "region_id": "foshan",
      "created_at": "2026-03-01T10:00:00Z"
    }
  }
}
```

**业务规则**：
- 若 `code` 换取的 `openid` 匹配已有设计师预创建记录（手机号匹配），则 `role` 返回 2。
- 新用户自动创建，`role` 默认为 1。
- `token` 有效期 7 天。

**错误码**：

| code | 场景 |
|:---|:---|
| 400 | `code` 无效或已过期 |
| 403 | 账号已被停用 |

**鉴权**：`PUBLIC`（价格字段根据角色动态返回）

**请求参数**（Query）：

| 参数 | 类型 | 必填 | 说明 |
|:---|:---|:---:|:---|
| `region_id` | String | ✓ | 区域 ID |
| `category_id` | Int | ✗ | 品类 ID |
| `space` | String | ✗ | 空间标签（如 `kitchen_bath`） |
| `scene` | String | ✗ | 场景标签（如 `minimalist`） |
| `keyword` | String | ✗ | 搜索关键词（匹配商品名称/品牌） |
| `sort_by` | String | ✗ | 排序字段：`default`(默认权重) / `price_asc` / `price_desc` / `newest` |
| `page` | Int | ✗ | 页码（默认 1） |
| `page_size` | Int | ✗ | 每页数量（默认 20，最大 50） |

**响应体**：

```json
{
  "code": 200,
  "data": {
    "list": [
      {
        "id": 10001,
        "name": "TOTO 浴室柜 800mm",
        "sku_code": "TOTO-YSG-800",
        "brand": "TOTO",
        "category_id": 3,
        "category_name": "洁具",
        "guide_price": 1280.00,
        "designer_price": 1088.00,
        "price_unit": "元/套",
        "cover_image": "https://cdn.baihou.life/product/10001/cover_400.webp",
        "space_tags": ["kitchen_bath"],
        "scene_tags": ["minimalist", "healing"]
      }
    ],
    "pagination": {
      "page": 1,
      "page_size": 20,
      "total": 56,
      "total_pages": 3
    }
  }
}
```

**业务规则**：
- `region_id` 为必传，未传时返回 400 错误。
- `guide_price`：游客返回 `null`，客户/设计师返回实际值。
- `designer_price`：仅 `role==2` 时返回，其他角色该字段不存在于响应中。
- `category_id`、`space`、`scene` 三个筛选条件为 AND 逻辑。
- 默认排序：`sort_order ASC, created_at DESC`。

### A06 意向留资

**POST** `/v1/leads/submit`

**鉴权**：`USER`

**请求体**：

| 参数 | 类型 | 必填 | 说明 |
|:---|:---|:---:|:---|
| `phone` | String | ✓ | 手机号（11位） |
| `name` | String | ✗ | 称呼 |
| `intention` | String | ✗ | 意向描述 |
| `product_id` | Long | ✗ | 关联商品 ID |
| `category_id` | Int | ✗ | 关注品类 ID |
| `region_id` | String | ✓ | 区域 ID |
| `source` | String | ✗ | 来源标识：`product_detail` / `home_banner` / `category_page` |

**响应体**：

```json
{
  "code": 200,
  "data": {
    "lead_id": 20001,
    "message": "已提交，我们将尽快联系您"
  }
}
```

**业务规则**：
- 同一用户 + 同一 `product_id` 24 小时内仅可提交一次，重复提交返回 409。
- `product_id` 为空时视为通用留资（非特定商品）。
- 手机号后端 AES 加密存储。

**错误码**：

| code | 场景 |
|:---|:---|
| 400 | 手机号格式不合法 |
| 409 | 24小时内重复留资 |

### A07 设计师素材下载 Token

**GET** `/v1/designer/asset-token`

**鉴权**：`DESIGNER`

**请求参数**（Query）：

| 参数 | 类型 | 必填 | 说明 |
|:---|:---|:---:|:---|
| `product_id` | Long | ✓ | 商品 ID |
| `media_id` | Long | ✓ | 媒体资源 ID |

**响应体**：

```json
{
  "code": 200,
  "data": {
    "download_url": "https://cdn.baihou.life/source/10001/TOTO_YSG800.skp?token=abc123&expires=1709370000",
    "file_name": "TOTO_YSG800.skp",
    "file_size": 15728640,
    "expires_at": "2026-03-02T10:05:00Z"
  }
}
```

**业务规则**：
- 校验当前用户 `role == 2` 且账号处于启用状态。
- 校验 `media_id` 对应文件的 `access_level` 包含 `designer`。
- 生成的 `download_url` 有效期 5 分钟。
- 每次成功调用写入下载日志（`designer_id`, `product_id`, `media_id`, `downloaded_at`, `ip`）。

**错误码**：

| code | 场景 |
|:---|:---|
| 403 | 非设计师角色 / 账号已停用 |
| 404 | 媒体资源不存在 |

### A08 创建预约

**POST** `/v1/appointment/create`

**鉴权**：`USER`

**优先级**：P1

**请求体**：

| 参数 | 类型 | 必填 | 说明 |
|:---|:---|:---:|:---|
| `customer_name` | String | ✓ | 姓名 |
| `customer_phone` | String | ✓ | 联系电话（11位） |
| `service_type` | String | ✓ | 服务类型枚举：`measure` / `delivery` / `install` / `after_sale` / `grout` |
| `preferred_date` | String | ✓ | 期望日期（格式 `YYYY-MM-DD`） |
| `preferred_time` | String | ✗ | 期望时段：`morning` / `afternoon` / `evening` |
| `address` | String | ✗ | 详细地址 |
| `remark` | String | ✗ | 备注 |
| `product_id` | Long | ✗ | 关联商品 |
| `region_id` | String | ✓ | 服务区域 |

**响应体**：

```json
{
  "code": 200,
  "data": {
    "appointment_id": 30001,
    "appointment_no": "APT202603020001",
    "status": "pending",
    "message": "预约已提交，我们将尽快与您确认"
  }
}
```

**业务规则**：
- `preferred_date` 不得早于今天。
- 生成预约单号格式：`APT` + `yyyyMMdd` + 4 位序号。
- 初始状态为 `pending`（待确认）。
- 创建后推送模板消息给对应区域服务团队。

**错误码**：

| code | 场景 |
|:---|:---|
| 400 | 日期格式错误 / 手机号格式不合法 |
| 422 | 期望日期早于今天 / 区域不支持该服务类型 |

### A10 创建订单

**POST** `/v1/order/create`

**鉴权**：`USER`

**优先级**：P1

**请求体**：

| 参数 | 类型 | 必填 | 说明 |
|:---|:---|:---:|:---|
| `region_id` | String | ✓ | 下单区域 |
| `items` | Array | ✓ | 订单项列表 |
| `items[].product_id` | Long | ✓ | 商品 ID |
| `items[].quantity` | Int | ✓ | 数量（>=1） |
| `remark` | String | ✗ | 备注 |

**响应体**：

```json
{
  "code": 200,
  "data": {
    "order_id": 40001,
    "order_no": "ORD2026030200000001",
    "total_amount": 2560.00,
    "pay_amount": 2560.00,
    "status": "pending_pay",
    "items": [
      {
        "product_id": 10001,
        "product_name": "TOTO 浴室柜 800mm",
        "sku_code": "TOTO-YSG-800",
        "unit_price": 1280.00,
        "quantity": 2,
        "subtotal": 2560.00
      }
    ],
    "expires_at": "2026-03-02T10:30:00Z"
  }
}
```

**业务规则**：
- 下单时按当前用户角色计算价格（客户=指导价，设计师=折扣价），价格快照到 OrderItem。
- 订单创建后状态为 `pending_pay`，30 分钟内未支付自动关闭。
- `order_no` 格式：`ORD` + `yyyyMMdd` + 8 位自增序号。
- 校验所有 `product_id` 在 `region_id` 范围内且处于上架状态。

**错误码**：

| code | 场景 |
|:---|:---|
| 422 | 商品已下架 / 商品不在该区域可售范围 / 数量非法 |

### A14 微信支付回调

**POST** `/v1/payment/notify`

**鉴权**：`PUBLIC`（微信服务器调用，通过签名验证身份）

**请求体**：微信标准支付通知报文（XML/JSON，按微信支付 V3 接口规范）

**处理逻辑**：

```
1. 验证微信签名（防伪造）
2. 解析通知报文，提取 order_no 和 transaction_id
3. 幂等判断：若订单已为 paid 状态，直接返回成功
4. 更新订单状态：pending_pay → paid
5. 写入 transaction_id 和 paid_at
6. 返回成功应答给微信（防止重复推送）
```

**响应体**（返回给微信）：

```json
{
  "code": "SUCCESS",
  "message": ""
}
```

**关键约束**：
- 此接口不通过 JWT 鉴权，改由微信签名验证。
- 需处理重复通知（幂等性保证）。
- 处理超时（5 秒内返回，否则微信会重试）。

# 第二部分 状态机

## 状态机通用说明

每个状态机定义：
- **状态枚举**：所有可能的状态值
- **转换规则**：允许从哪个状态转到哪个状态
- **触发条件**：什么操作/事件触发转换
- **操作方**：谁有权触发（前台/后台/系统定时任务）

---

## S1 用户状态机

### 状态枚举

| 状态值 | 名称 | 说明 |
|:---|:---|:---|
| `anonymous` | 游客 | 未登录状态 |
| `active_customer` | 已激活客户 | 已登录的普通客户 |
| `active_designer` | 已激活设计师 | 已授权的设计师 |
| `disabled` | 已停用 | 账号被管理员停用 |

### 状态转换图

```
                 ┌──────────────────────────┐
                 │                          │
                 ▼                          │
┌──────────┐  登录成功  ┌──────────────────┐ │
│ anonymous │ ───────▶ │ active_customer   │ │
│  (游客)    │          │  (已激活客户)       │ │
└──────────┘          └───────┬──────────┘ │
                              │             │
                     后台关联设计师记录         │ Token过期/
                              │             │ 主动退出
                              ▼             │
                     ┌──────────────────┐   │
                     │ active_designer   │ ──┘
                     │  (已激活设计师)      │
                     └───────┬──────────┘
                              │
                      管理员停用 │ 管理员启用
                              ▼
                     ┌──────────────────┐
                     │    disabled       │
                     │   (已停用)         │
                     └──────────────────┘
```

### 转换规则表

| 当前状态 | 目标状态 | 触发条件 | 操作方 |
|:---|:---|:---|:---|
| `anonymous` | `active_customer` | 微信登录成功 | 系统（自动） |
| `anonymous` | `active_designer` | 微信登录且手机号匹配预创建设计师记录 | 系统（自动） |
| `active_customer` | `active_designer` | 管理员后台创建设计师并关联该手机号 | 后台管理员 |
| `active_customer` | `anonymous` | Token 过期 / 主动退出登录 | 前台 / 系统 |
| `active_designer` | `anonymous` | Token 过期 / 主动退出登录 | 前台 / 系统 |
| `active_customer` | `disabled` | 管理员停用账号 | 后台管理员 |
| `active_designer` | `disabled` | 管理员停用账号 | 后台管理员 |
| `disabled` | `active_customer` | 管理员重新启用 | 后台管理员 |
| `disabled` | `active_designer` | 管理员重新启用（若有设计师记录） | 后台管理员 |

## S2 商品状态机

### 状态枚举

| 状态值 | 名称 | 说明 |
|:---|:---|:---|
| `draft` | 草稿 | 新建未上架，前台不可见 |

```

│ (草稿)  │           │ (已上架)   │
└────┬───┘           └──┬────┬──┘
     │                  │    │
     │    ┌─────────────┘    │
     │    │ 下架              │ 归档
     │    ▼                  │
     │  ┌──────────┐         │
     │  │ off_shelf │         │
     │  │ (已下架)   │         │
     │  └──┬────┬──┘         │
     │     │    │             │
     │     │    │ 重新上架      │
     │     │    └─────────────┤
     │     │ 归档              │
     │     ▼                  ▼
     │  ┌──────────────────────┐
     └─▶│      archived        │
        │      (已归档)          │
        └──────────────────────┘
```

### 转换规则表

| 当前状态 | 目标状态 | 触发条件 | 操作方 |
|:---|:---|:---|:---|
| `draft` | `on_shelf` | 管理员点击"发布上架" | 后台管理员 |
| `draft` | `archived` | 管理员删除草稿 | 后台管理员 |
| `on_shelf` | `off_shelf` | 管理员手动下架 | 后台管理员 |
| `on_shelf` | `off_shelf` | 关联品类被停用（系统自动） | 系统 |
| `on_shelf` | `archived` | 管理员直接归档 | 后台管理员 |
| `off_shelf` | `on_shelf` | 管理员重新上架 | 后台管理员 |
| `off_shelf` | `archived` | 管理员归档 | 后台管理员 |

**业务约束**：
- `archived` 为终态，不可恢复。
- 上架要求：商品必须有 `name`、`sku_code`、`category_id`、至少一个 `region`、至少一张图片。
- 批量操作（B01-6）遵循相同的转换规则。

## S3 预约状态机

### 状态枚举

| 状态值 | 名称 | 说明 |
|:---|:---|:---|
| `pending` | 待确认 | 客户提交，等待后台确认 |
| `confirmed` | 已确认 | 后台已确认，等待上门 |
| `in_progress` | 进行中 | 服务人员已上门，正在服务 |
| `completed` | 已完成 | 服务完成 |
| `cancelled` | 已取消 | 预约取消（客户或后台） |

### 状态转换图

```
                   客户取消
┌──────────┐  ─────────────────────────────┐
│  pending  │                              │
│ (待确认)  │──┐                            ▼
└──┬───────┘  │ 后台拒绝          ┌──────────────┐
   │          └──────────────────▶│  cancelled    │
   │ 后台确认                      │  (已取消)      │
   ▼                              └──────────────┘
┌──────────┐                              ▲
│ confirmed │                              │
│ (已确认)  │──┐ 客户取消                     │
└──┬───────┘  └───────────────────────────┘
   │
   │ 开始服务
   ▼
┌──────────────┐
│ in_progress   │
│ (进行中)       │
└──┬───────────┘
   │
   │ 服务完成
   ▼
┌──────────────┐
│  completed    │
│  (已完成)      │
└──────────────┘
```

### 转换规则表

| 当前状态 | 目标状态 | 触发条件 | 操作方 |
|:---|:---|:---|:---|
| `pending` | `confirmed` | 后台确认预约 | 后台管理员 |
| `pending` | `cancelled` | 客户取消预约 | 前台用户 |
| `pending` | `cancelled` | 后台拒绝预约 | 后台管理员 |
| `confirmed` | `in_progress` | 服务人员开始服务（后台更新） | 后台管理员 |
| `confirmed` | `cancelled` | 客户取消（确认后24小时内） | 前台用户 |
| `confirmed` | `cancelled` | 后台取消 | 后台管理员 |
| `in_progress` | `completed` | 服务完成 | 后台管理员 |

**业务约束**：
- `completed` 和 `cancelled` 为终态。
- 客户仅可在 `pending` 和 `confirmed` 状态取消，`in_progress` 不可取消。
- 确认后超过24小时的客户取消需电话确认（前端提示"请拨打客服电话取消"）。
- 状态变更后推送模板消息通知客户。

## S4 订单状态机

### 状态枚举

| 状态值 | 名称 | 说明 |
|:---|:---|:---|
| `pending_pay` | 待支付 | 订单已创建，等待支付 |
| `paid` | 已支付 | 支付成功，等待处理 |
| `processing` | 处理中 | 商家确认并处理订单 |
| `shipped` | 已发货 | 商品已发出 |
| `completed` | 已完成 | 客户确认收货 |
| `closed` | 已关闭 | 订单关闭（超时/取消） |

### 状态转换图

```
┌──────────────┐
│ pending_pay   │
│ (待支付)       │
└──┬────────┬──┘
   │        │
   │支付成功  │ 超时(30min)/
   │        │ 客户取消
   ▼        ▼
┌────────┐ ┌──────────┐
│  paid   │ │  closed   │
│(已支付) │ │ (已关闭)   │
└──┬─────┘ └──────────┘
   │              ▲
   │ 后台确认处理   │
   ▼              │
┌────────────┐    │
│ processing  │    │
│ (处理中)     │────┘ 后台关闭(异常)
└──┬─────────┘
   │
   │ 发货
   ▼
┌────────────┐
│  shipped    │
│ (已发货)     │
└──┬─────────┘
   │
   │ 确认收货(客户/自动15天)
   ▼
┌────────────┐
│ completed   │
│ (已完成)     │
└────────────┘
```

### 转换规则表

| 当前状态 | 目标状态 | 触发条件 | 操作方 |
|:---|:---|:---|:---|
| `pending_pay` | `paid` | 微信支付回调确认 | 系统（自动） |
| `pending_pay` | `closed` | 超过 30 分钟未支付 | 系统（定时任务） |
| `pending_pay` | `closed` | 客户主动取消 | 前台用户 |
| `paid` | `processing` | 后台确认订单 | 后台管理员 |
| `paid` | `closed` | 后台关闭（异常情况，需退款） | 后台管理员 |
| `processing` | `shipped` | 后台确认发货，填写物流单号 | 后台管理员 |
| `processing` | `closed` | 后台关闭（异常情况，需退款） | 后台管理员 |
| `shipped` | `completed` | 客户确认收货 | 前台用户 |
| `shipped` | `completed` | 发货后 15 天自动确认 | 系统（定时任务） |

**业务约束**：
- `completed` 和 `closed` 为终态。
- 仅 `pending_pay` 状态可由客户取消。
- `paid` → `closed` 和 `processing` → `closed` 场景需后台同步发起退款（一期暂走线下退款）。
- 定时任务：
  - 每分钟扫描 `pending_pay` 超 30 分钟的订单，自动关闭。
  - 每天扫描 `shipped` 超 15 天的订单，自动确认完成。

## S5 线索状态机

### 状态枚举

| 状态值 | 名称 | 说明 |
|:---|:---|:---|
| `new` | 未跟进 | 新建线索，尚未分配或跟进 |
| `following` | 跟进中 | 已分配销售，正在跟进 |
| `converted` | 已成单 | 线索转化为订单/预约 |
| `abandoned` | 已放弃 | 线索无效或客户放弃 |

### 状态转换图

```
┌──────────┐
│   new     │
│ (未跟进)  │
└──┬───────┘
   │
   │ 分配销售/开始跟进
   ▼
┌──────────┐
│ following │
│ (跟进中)  │
└──┬────┬──┘
   │    │
   │    │ 客户无意向/无效
   │    ▼
   │  ┌──────────┐
   │  │ abandoned │
   │  │ (已放弃)   │
   │  └──────────┘
   │
   │ 客户下单/预约成功
   ▼
┌──────────┐
│ converted │
│ (已成单)   │
└──────────┘
```

### 转换规则表

| 当前状态 | 目标状态 | 触发条件 | 操作方 |
|:---|:---|:---|:---|
| `new` | `following` | 分配给销售人员 / 销售开始跟进 | 后台管理员 / 自动分配 |
| `new` | `abandoned` | 管理员判定无效线索 | 后台管理员 |
| `following` | `converted` | 客户完成下单或预约 | 后台管理员 |
| `following` | `abandoned` | 跟进后客户明确放弃 / 无法联系 | 后台管理员 |
| `abandoned` | `following` | 重新激活跟进（客户回头） | 后台管理员 |

**业务约束**：
- `converted` 为终态，不可变更。
- `abandoned` 可回退到 `following`（客户可能回头）。
- 线索创建后若 48 小时未分配，系统自动提醒管理员。
- 跟进中线索超过 7 天无操作，系统自动标记提醒。
