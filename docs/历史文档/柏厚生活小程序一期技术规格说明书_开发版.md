# 柏厚生活小程序一期技术规格说明书 (V1.0)

**面向对象**：前端、后端、测试工程师
**核心目标**：实现全案空间展示、设计师素材赋能、区域化选品功能。

---

## 1. 核心模型与数据结构

### 1.1 商品 (Product)

| 字段 | 类型 | 说明 |
| :--- | :--- | :--- |
| `id` | Long | 主键 |
| `name` | String | 商品名称 |
| `sku_code` | String | 唯一SKU编码 |
| `category_id` | Int | 关联34个品类ID（瓷砖、橱柜等） |
| `guide_price` | Decimal | 指导价（普通用户可见） |
| `designer_discount`| Float | 设计师折扣率（如 0.85） |
| `regions` | Array | 可售区域ID列表（如 ["foshan", "chengdu"]） |
| `space_tags` | Array | 空间标签（玄关、客厅、卧室等） |
| `scene_tags` | Array | 场景标签（亲子、收纳、极简等） |
| `spec_params` | JSON | 规格参数（材质、尺寸、颜色等） |

### 1.2 媒体资源 (Media)

| 字段 | 类型 | 说明 |
| :--- | :--- | :--- |
| `scene_images` | Array | 场景效果图预览地址 |
| `element_images` | Array | 产品元素图（透明底或白底图） |
| `source_file_url` | String | 设计师专用原文件下载地址（如 .max, .skp） |

---

## 2. 核心功能及实现逻辑

### 2.1 身份认证与价格逻辑

- **认证方式**：微信一键登录（手机号授权）。
- **角色标识**：`role` (1:普通客户, 2:设计师)。设计师账号由后台管理员手动创建或审核。
- **UI逻辑**：
  - `role == 1`：显示 `guide_price`；隐藏“素材下载”按钮。
  - `role == 2`：计算显示 `designer_price` (`guide_price * designer_discount`)；显示“素材下载”按钮。

### 2.2 区域化展示逻辑 (Regionalization)

- **获取定位**：小程序启动时请求 `wx.getLocation`。
- **全局状态**：Store 维护 `currentRegionId`。
- **查询过滤**：所有产品列表、分类接口均需显式传入 `region_id`，后端仅返回该区域可见的商品。

### 2.3 选品中心 (Product Center)

- **多轴筛选**：
  - 维度1：品类 (Category Tree)
  - 维度2：空间 (Space)
  - 维度3：场景 (Scene)
- **技术点**：后端需要支持对这三个维度的 `AND` 逻辑组合查询。

### 2.4 设计师素材分发

- **下载安全**：`source_file_url` 建议通过后端接口获取带签名的临时 URL (Token-based)。
- **前端实现**：利用 `wx.downloadFile` 下载后，调用 `wx.openDocument` 或提示复制下载链接。

## 3. 接口需求摘要 (APIs)

### 3.1 客户端/小程序端

- `GET /v1/config/regions`：获取支持的城市列表及基准定位。
- `POST /v1/auth/wechat-login`：微信登录及身份返回。
- `GET /v2/product/list`：分页查询商品（支持 region/cat/space/scene 过滤）。
- `GET /v2/product/{id}`：商品详情，含参数及媒体资源。
- `POST /v1/leads/submit`：客户意向留资。
- `GET /v1/user/designer-assets`：获取专属素材下载Token（限设计师角色）。

### 3.2 管理后端 (CMS)

- 品类树管理 (支持34个层级分类配置)。
- 商品管理 (包含区域映射、标签挂载、原文件上传)。
- 设计师管理 (账号创建、改价系数配置)。
- 线索查看 (时间/城市/品类维度统计导出)。

---

## 4. 非功能要求

- **图片优化**：商品图必须接入 CDN，支持 WebP 格式，避免大图导致的小程序卡顿。
- **性能**：商品列表页首屏响应时间需控制在 1s 以内（需做索引优化）。
- **安全**：严防非设计师角色通过爬虫接口抓取 `source_file_url`。
