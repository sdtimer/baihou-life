# 柏厚生活小程序 — 首页优化方案（V2.0）

**文档编号**：BHS-OPT-20260317-V1
**编制依据**：03-17 小程序设计与数据安排会议决议
**版本**：V1.0
**编制日期**：2026-03-17

---

## 1. 优化背景

### 1.1 现有方案痛点

当前首页设计存在以下问题（会议反馈汇总）：

1. **信息过载**：首页顶部直接暴露品类/空间/场景三轴选择器，终端用户进入后不知从何下手
2. **视觉吸引力不足**：产品图片偏小，无法第一时间传递品牌质感
3. **用户角色未区分**：设计师和终端用户共用同一套浏览逻辑，各自需求未被有效满足
4. **功能优先于体验**：页面偏工具化，缺乏"逛"的感觉，与家居行业用户的浏览习惯不匹配

### 1.2 优化目标

- 降低首页认知负担，让用户 3 秒内产生浏览兴趣
- 对齐小红书式内容消费习惯，提升停留时长
- 1.0 聚焦单品展示，为 2.0 空间展示预留扩展点
- 保持与现有后端标签/分类系统的兼容性

## 2. 首页架构重设计

### 2.1 页面层级调整

**调整前（当前版本）**：
```
首页 = 定位栏 + 品类/空间/场景选择器 + 产品网格列表
```

**调整后（优化版本）**：
```
Splash 品牌过渡页（设计提供，停留1-2秒）→ 首页
首页 = 品牌形象区 + 搜索栏 + 内容信息流（卡片瀑布流）
选品中心 = 品类/空间/场景筛选 + 产品网格列表（移至独立Tab页）
```

### 2.2 Splash 品牌过渡页

- 由设计提供视觉稿，展示品牌 Logo + 核心 Slogan + 公司简介
- 停留约 1-2 秒后自动跳转首页
- 期间可异步完成定位获取、登录态检查等初始化操作

### 2.3 首页结构（自上而下）

```
┌──────────────────────────────┐
│  ① 顶部状态栏 + 搜索入口       │   固定吸顶
├──────────────────────────────┤
│  ② 品牌形象区                  │   品牌 Logo + Slogan + 城市定位
│     - 公司简介（首屏下滑可见）    │   满足洪总要求的公司介绍展示
├──────────────────────────────┤
│  ③ 快捷入口栏（可选）           │   2-4个图标入口：全部产品/空间/新品/设计师
├──────────────────────────────┤
│  ④ 内容信息流（核心区域）        │   小红书式双列瀑布流卡片
│     - 精选搭配案例               │   大图 + 标题 + 标签
│     - 爆款单品推荐               │   产品图 + 名称 + 价格
│     - 小红书已发内容复用          │   效果图/软装搭配
│     - 场景灵感（预留）           │   为2.0空间展示铺垫
│                                 │
│        ┌─────┐ ┌─────┐        │
│        │ 卡片 │ │ 卡片 │        │
│        │     │ │     │        │
│        └─────┘ └─────┘        │
│        ┌─────┐ ┌─────┐        │
│        │ 卡片 │ │ 卡片 │        │
│        └─────┘ └─────┘        │
│          ...下拉加载更多          │
├──────────────────────────────┤
│  ⑤ 底部导航栏                   │   首页 / 选品 / 收藏 / 我的
└──────────────────────────────┘
```

### 2.4 底部导航栏（TabBar）设计

| Tab | 名称 | 功能 |
|:---|:---|:---|
| Tab 1 | 发现 | 小红书式内容信息流首页 |
| Tab 2 | 选品 | 品类/空间/场景三轴筛选 + 产品列表（原首页核心功能迁移至此） |
| Tab 3 | 收藏 | 用户收藏的产品/内容 |
| Tab 4 | 我的 | 个人中心、订单、预约、设计师入口 |

## 3. 内容卡片设计规范

### 3.1 卡片类型定义

| 卡片类型 | 内容来源 | 展示元素 | 点击行为 |
|:---|:---|:---|:---|
| **产品卡** | 后台商品数据 | 产品主图 + 名称 + 价格（按角色） | 跳转产品详情页 |
| **案例卡** | 运营配置/小红书内容 | 效果图 + 标题 + 风格标签 | 跳转案例详情（图文） |
| **场景卡** | 空间组合图（2.0预留） | 空间效果图 + 空间名称 | 跳转空间详情（2.0上线后启用） |
| **软文卡** | 运营图文内容 | 封面图 + 标题 + 摘要 | 跳转图文详情页 |

### 3.2 卡片视觉规范

- 双列瀑布流布局，卡片宽度各占 `50% - 间距`
- 图片比例不固定（允许竖图和横图混排），最小高度 180rpx
- 图片圆角 16rpx，卡片底部留白展示文字信息
- 标签使用胶囊样式，最多展示 2 个标签
- 价格展示遵循角色权限（游客隐藏、客户显示指导价、设计师显示折扣价）

### 3.3 信息流排序策略

1. 运营置顶内容（后台可配置权重）
2. 本地区域关联产品/案例优先
3. 新品优先
4. 基于浏览行为的个性化推荐（2.0预留）

## 4. 选品中心页（原首页功能迁移）

将原首页的品类/空间/场景筛选能力完整迁移到「选品」Tab 页：

### 4.1 页面结构

```
┌──────────────────────────────┐
│  搜索栏                        │
├──────────────────────────────┤
│  品类 / 空间 / 场景 切换Tab     │   三轴筛选保留，作为此页核心
├──────────────────────────────┤
│  筛选条件展示区                  │   已选标签 + 清空
├──────────────────────────────┤
│  产品网格列表                    │   双列/单列可切换
│                                 │   每行展示更大图片（满足洪总要求）
└──────────────────────────────┘
```

### 4.2 优化点

- **品类列表全展开**：不使用推拉/折叠，直接列出所有品类（会议明确要求），可增加行数
- **图片放大**：单品图片尺寸增大，宽度至少 340rpx，高度自适应
- **空间入口默认折叠**：空间Tab保留但标记"即将上线"，有内容后展开

## 5. 数据与内容准备方案

### 5.1 一期内容数据清单

| 数据类型 | 数量 | 来源 | 状态 | 优先级 |
|:---|:---|:---|:---|:---|
| 瓷砖产品 | 百度云盘全量 | 柏厚瓷砖数据包 | 待录入 | P0 本周完成 |
| 洁具产品 | ~21款 | 陈总提供 | 陆续提供中 | P0 |
| 软装组合图 | 4/14 套 | 设计部门制作 | 制作中 | P1 |
| 产品主图 | 按SKU数 | 供应商/京东抓取 | 待整理 | P0 |
| 小红书内容 | 已发内容 | 小红书搬运 | 可用 | P1 |
| 产品文案 | 瓷砖有基础文案 | 自有/补写 | 部分缺失 | P1 |

### 5.2 数据录入流程

```
百度云盘数据包
    ↓
解压整理：按系列/品类归类
    ↓
后台录入：产品基本信息（名称、编号、规格、系列名）
    ↓
图片上传：产品主图 + 详情图上传至 OSS
    ↓
标签配置：品类标签 + 空间标签（先大类，后续细化）
    ↓
审核上架
```

### 5.3 分工建议

| 任务 | 负责人 | 协助 | 预计工时 |
|:---|:---|:---|:---|
| 瓷砖数据批量录入 | 小明 | 分散给3-4人并行 | 2-3天 |
| 瓷砖分类标签配置 | 小明 + 敏芝 | - | 1天 |
| 洁具数据整理 | 陈总 | - | 本周内 |
| 首页卡片内容配置后台 | 黄总 | - | 随首页改版 |
| 小红书内容搬运 | 运营 | - | 持续 |

## 6. 技术改动评估

### 6.1 前端改动

| 改动项 | 影响范围 | 工作量评估 |
|:---|:---|:---|
| Splash 品牌过渡页 | `pages/splash/` 新增 | 0.5-1天 |
| 首页重构为瀑布流 | `pages/home/` 整体重写 | 3-5天 |
| 新增内容卡片组件 | `components/content-card/` | 1-2天 |
| TabBar 调整（新增"选品"Tab） | `app.json` + 新页面 | 1天 |
| 选品中心页（从首页迁移） | `pages/product/center/` | 1-2天 |
| 品类列表全展开（去推拉） | `pages/product/list/` | 0.5天 |

### 6.2 后端改动

| 改动项 | 影响范围 | 工作量评估 |
|:---|:---|:---|
| 内容信息流接口 | 新增 `/v1/feed/list` | 2-3天 |
| 内容管理后台（CMS 扩展） | 新增内容类型管理 | 2-3天 |
| 首页推荐位配置调整 | 调整 Banner → 卡片推荐位 | 1天 |
| 空间入口开关配置 | 后台增加开关字段 | 0.5天 |

### 6.3 与现有系统兼容性

- 标签系统：**完全兼容**，后端已支持灵活标签，无需改动
- 角色权限：**完全兼容**，价格展示逻辑不变
- 区域化：**完全兼容**，定位推荐逻辑不变
- 商品数据结构：**无需改动**，卡片展示使用已有商品字段

## 8. 风险与应对

| 风险 | 影响 | 应对措施 |
|:---|:---|:---|
| 内容素材不足，首页信息流显得空 | 用户体验差 | 前期用产品卡填充 + 小红书内容搬运 + 运营产出案例内容 |
| 产品图片质量参差不齐 | 品牌形象受损 | 统一图片规格要求，劣质图片不上架 |
| 瓷砖数据录入工作量大 | 影响上线节奏 | 多人分工并行，优先录入重点系列 |
| 空间展示延后，洪总期望管理 | 业务方不满 | 明确 2.0 路线图，1.0 先用优质单品 + 案例图过渡 |
| 首页改版影响已有页面逻辑 | 开发返工 | 选品中心页复用原首页核心组件，减少重复开发 |
