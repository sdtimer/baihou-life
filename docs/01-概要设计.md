## 1. 项目概述

### 1.1 项目背景

柏厚生活定位为"空间场景家居领先品牌"，业务覆盖成都、佛山、武汉等多个城市。一期小程序旨在快速落地"卖空间、卖场景、卖全案、卖交付"的品牌理念，搭建线上选品与引流阵地。

### 1.2 项目目标

- 实现全案空间展示、设计师素材赋能、区域化选品功能
- 覆盖 34 项核心品类，支持品类 / 空间 / 场景三轴筛选
- 为 C 端业主提供本地化产品展示与意向留资
- 为 B 端设计师提供专属素材下载与折扣价查看
- 后台 CMS 支撑商品管理、品类维护、线索跟进等运营能力

### 1.3 核心用户角色

| 角色 | role 值 | 来源 | 能力边界 |
|:---|:---|:---|:---|
| 游客 | 0 | 未登录 | 浏览首页、产品列表（不含价格） |
| 普通客户 | 1 | 微信授权登录 | 查看指导价、提交留资/预约、下单支付 |
| 设计师 | 2 | 后台管理员创建 | 查看设计师折扣价、下载素材原文件 |
| 管理员 | 99 | 系统预置 | CMS 全部操作权限 |

## 2. 技术选型

### 2.1 总体技术栈

| 层次 | 技术选型 | 说明 |
|:---|:---|:---|
| **前端（小程序）** | 微信小程序原生 / uni-app | 微信生态内运行，支持 wx.login / wx.requestPayment 等原生 API |
| **后端框架** | RuoYi（若依）— Spring Boot 版 | 基于若依二次开发，复用其权限管理、代码生成、日志审计等基础能力 |
| **数据库** | MySQL 8.0 | 若依默认数据库，使用 JSON 类型字段存储标签/区域数组 |
| **缓存** | Redis | 会话管理、Token 缓存、热点数据缓存（品类树、区域列表） |
| **对象存储** | 阿里云 OSS / 腾讯云 COS | 商品图片、设计师素材文件存储，接入 CDN 分发 |
| **支付** | 微信支付 V3 | 小程序内嵌支付，JSAPI 模式 |
| **消息推送** | 微信模板消息 | 预约/订单状态变更通知 |

### 2.2 若依框架复用说明

| 若依模块 | 复用方式 |
|:---|:---|
| `ruoyi-system` | 复用系统用户、角色、菜单、部门管理，管理员直接使用若依后台登录 |
| `ruoyi-common` | 复用通用工具类、分页组件、文件上传、Excel 导出 |
| `ruoyi-framework` | 复用安全框架（Spring Security）、Token 鉴权、跨域配置 |
| `ruoyi-generator` | 利用代码生成器快速生成商品、品类、订单等业务表的 CRUD 代码 |
| `ruoyi-quartz` | 复用定时任务框架，用于订单超时关闭、发货自动确认等 |

**新增业务模块**：在若依工程中新增 `ruoyi-baihou` 模块，承载所有柏厚生活业务代码（商品、订单、线索、预约、小程序认证等）。

### 2.3 小程序端技术方案

| 方面 | 方案 |
|:---|:---|
| 开发框架 | 微信小程序原生（WXML + WXSS + JS），或基于 uni-app 开发 |
| 状态管理 | 全局 Store（MobX-miniprogram 或 Pinia，视框架选择） |
| 网络请求 | 封装统一请求工具，自动携带 Token 和 region_id |
| 图片加载 | CDN + WebP 自适应 + lazy-load |
| 定位 | `wx.getLocation({ type: 'gcj02' })`，Haversine 公式匹配最近城市 |

## 5. 关键业务流程

### 5.1 价格展示逻辑

```
游客 (role=0)    → 显示"登录查看价格"
普通客户 (role=1) → 显示 guide_price（指导价）
设计师 (role=2)  → 显示 guide_price（划线价）+ designer_price（专属价，红色高亮）
                   designer_price = guide_price × designer_discount（后端计算返回）
```

### 5.2 区域化筛选逻辑

- 所有产品查询接口必须传入 `region_id`
- 后端过滤逻辑：商品 `regions` 字段包含 `"ALL"` 或包含当前 `region_id`
- 区域切换后清空产品列表缓存，重新拉取

### 5.3 三轴筛选

品类（category_id）、空间（space）、场景（scene）三个维度采用 AND 逻辑组合查询，后端通过 MySQL JSON 函数实现标签匹配。

## 6. 非功能性要求

| 指标 | 要求 |
|:---|:---|
| 首屏响应 | 商品列表页 < 1s |
| 图片优化 | CDN + WebP 自适应，列表页 400px 缩略图，详情页 1200px |
| 安全 | JWT Token 鉴权、手机号 AES 加密存储、素材下载 Token 签名（5分钟过期） |
| 可扩展性 | 品类支持后台灵活增删改、商品区域/标签可动态配置 |
| 定时任务 | 订单 30 分钟未支付自动关闭、发货 15 天自动确认收货、线索 48 小时未分配提醒 |
