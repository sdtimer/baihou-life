## 六、下午追加工作：商品管理 & Banner 管理模块改造

### 背景

对照 `docs/product-module-refactor.md` 和 `docs/banner-module-refactor.md` 两份设计文档，对商品管理、Banner 管理、品类管理三个模块进行功能层改造。核心目标是将原来的纯文本 JSON 字段替换为有约束、有体验的交互组件。

---

### 改造内容

#### 1. 区域 options 接口（后端）

在 `BaihouRegionMapper` / `IBaihouRegionService` / `BaihouRegionServiceImpl` 上追加 `selectActiveOptions()` 方法，返回 `{value, label}` 格式的启用区域列表。`BaihouRegionController` 新增路由：

```
GET /admin/regions/options   权限: baihou:region:list
```

对应前端 `api/baihou/regions.js` 追加 `getRegionOptions()` 函数。

#### 2. 规格参数模板（后端）

新建完整的 spec def 垂直切面（实体 → Mapper → Service → Controller）：

- **`BaihouCategorySpecDef.java`**：字段 specDefId / categoryId / specKey / specLabel / inputType / options / unit / isRequired / sortOrder，继承 BaseEntity
- **`BaihouCategorySpecDefMapper.java` + xml**：CRUD 4 个方法，xml 含 ResultMap 和完整 SQL
- **`IBaihouCategorySpecDefService.java` + `BaihouCategorySpecDefServiceImpl.java`**：4 个 service 方法
- **`BaihouCategoryController.java`** 追加 4 个路由：

```
GET    /admin/categories/{categoryId}/spec-defs
POST   /admin/categories/{categoryId}/spec-defs
PUT    /admin/categories/{categoryId}/spec-defs/{specDefId}
DELETE /admin/categories/{categoryId}/spec-defs/{specDefId}
权限均复用 baihou:category:*
```

对应前端 `api/baihou/categories.js` 追加 `listSpecDefs / addSpecDef / updateSpecDef / removeSpecDef` 4 个函数。

#### 3. SQL 初始化脚本

新建 `sql/baihou/12_spec_dict_init.sql`，内容：

- `bh_category_spec_def` 建表 DDL（含索引 `idx_category_id`）
- 插入 `sys_dict_type`：`baihou_space_tag`（空间标签）、`baihou_scene_tag`（场景标签）
- 插入 `sys_dict_data`：空间标签 9 条（客厅/餐厅/卧室/书房/儿童房/厨房/卫生间/阳台/玄关）、场景标签 9 条（现代简约/北欧/奶油/中古/新中式/轻奢/美式/工业风/法式）
- dictValue 使用**中文**，与存量商品 JSON 字段完全兼容，无需数据迁移

#### 4. 商品管理页改造（`views/baihou/products/index.vue`）

- **区域字段**：原 `el-input` 纯文本 → `el-select multiple`，选项来自 `getRegionOptions()`；新增 `regionsArr` ref，handleEdit 时 parse JSON 回填，submitForm 前序列化；筛选栏区域下拉同步改用 `regionSelectOptions`；验证规则改绑 `regionsArr`，trigger 改为 `change`
- **空间/场景标签**：原 `el-input` 纯文本 → `el-select multiple`，选项分别来自 `proxy.getDicts('baihou_space_tag')` 和 `proxy.getDicts('baihou_scene_tag')`；新增 `spaceTagsArr` / `sceneTagsArr` ref
- **规格参数动态表单**：新增 `specDefs` ref 和 `specParamsObj` ref；watch `form.categoryId`，变化时调用 `listSpecDefs(categoryId)` 加载模板、初始化 specParamsObj；模板 v-for 按 inputType 渲染 el-input / el-input-number / el-select；handleEdit 时 parse specParams 回填，submitForm 前序列化

#### 5. Banner 管理页改造（`views/baihou/banners/index.vue`）

- **投放区域字段**：原 `el-input` 纯文本 → `el-select multiple`，含”全部区域”（value=`ALL`）特殊选项和下方”全选”/”清空”快捷按钮
- **ALL 互斥逻辑**：`handleRegionChange` 中实现：选 ALL 时清除具体区域，选具体区域时清除 ALL；”全选”直接赋值 `[“ALL”]`
- **表格区域列**：原纯文本 → Tag 展示，`ALL` 使用 `type=primary`，具体区域使用 `type=info`，通过 `parseRegionTags()` 函数映射区域名称

#### 6. 品类管理页改造（`views/baihou/categories/index.vue`）

- 操作列新增”规格模板”按钮（`v-hasPermi=”['baihou:category:edit']”`）
- 新增规格模板主对话框：列出该品类的规格定义列表（specLabel/specKey/inputType/unit/isRequired/sortOrder），支持新增、编辑、删除
- 新增规格模板编辑内嵌子对话框：表单字段含显示名称/字段Key/输入类型（文本/数字/下拉选择）/选项（select类型条件渲染）/单位/必填/排序
- 规格模板状态与品类列表解耦，维护独立的 `specDefList` / `specDefForm` / `specDefOpen` 状态

### 关键设计约定

| 约定点 | 决策 |
|--------|------|
| dictValue 格式 | 中文（如 `”客厅”`），与存量 JSON 兼容 |
| regions 存储 | 不改表结构，仍 JSON 字符串，前端 parse/serialize |
| ALL 语义 | `[“ALL”]` 表示投放所有区域，与具体区域互斥 |
| specParams 存储 | 不改 `bh_product` 表结构，继续存 JSON |
| 后端验证 | 不做区域合法性服务端校验，前端下拉已限制范围 |
