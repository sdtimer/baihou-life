### 背景

对照 `docs/product-module-refactor.md` 和 `docs/banner-module-refactor.md` 两份设计文档，对商品管理、Banner 管理、品类管理三个模块进行功能层改造。核心目标是将原来的纯文本 JSON 字段替换为有约束、有体验的交互组件。

---

### 改造内容

#### 1. 区域 options 接口（后端）

在 `BaihouRegionMapper` / `IBaihouRegionService` / `BaihouRegionServiceImpl` 上追加 `selectActiveOptions()` 方法，返回 `{value, label}` 格式的启用区域列表。`BaihouRegionController` 新增路由：

```
GET /admin/regions/options   权限: baihou:region:list
```

对应前端 `api/baihou/regions.js` 追加 `getRegionOptions()` 函数。

#### 2. 规格参数模板（后端）

新建完整的 spec def 垂直切面（实体 → Mapper → Service → Controller）：

- **`BaihouCategorySpecDef.java`**：字段 specDefId / categoryId / specKey / specLabel / inputType / options / unit / isRequired / sortOrder，继承 BaseEntity
- **`BaihouCategorySpecDefMapper.java` + xml**：CRUD 4 个方法，xml 含 ResultMap 和完整 SQL
- **`IBaihouCategorySpecDefService.java` + `BaihouCategorySpecDefServiceImpl.java`**：4 个 service 方法
- **`BaihouCategoryController.java`** 追加 4 个路由：

```
GET    /admin/categories/{categoryId}/spec-defs
POST   /admin/categories/{categoryId}/spec-defs
PUT    /admin/categories/{categoryId}/spec-defs/{specDefId}
DELETE /admin/categories/{categoryId}/spec-defs/{specDefId}
权限均复用 baihou:category:*
```

对应前端 `api/baihou/categories.js` 追加 `listSpecDefs / addSpecDef / updateSpecDef / removeSpecDef` 4 个函数。

#### 3. SQL 初始化脚本

新建 `sql/baihou/12_spec_dict_init.sql`，内容：

- `bh_category_spec_def` 建表 DDL（含索引 `idx_category_id`）
- 插入 `sys_dict_type`：`baihou_space_tag`（空间标签）、`baihou_scene_tag`（场景标签）
- 插入 `sys_dict_data`：空间标签 9 条（客厅/餐厅/卧室/书房/儿童房/厨房/卫生间/阳台/玄关）、场景标签 9 条（现代简约/北欧/奶油/中古/新中式/轻奢/美式/工业风/法式）
- dictValue 使用**中文**，与存量商品 JSON 字段完全兼容，无需数据迁移

#### 4. 商品管理页改造（`views/baihou/products/index.vue`）

- **区域字段**：原 `el-input` 纯文本 → `el-select multiple`，选项来自 `getRegionOptions()`；新增 `regionsArr` ref，handleEdit 时 parse JSON 回填，submitForm 前序列化；筛选栏区域下拉同步改用 `regionSelectOptions`；验证规则改绑 `regionsArr`，trigger 改为 `change`
- **空间/场景标签**：原 `el-input` 纯文本 → `el-select multiple`，选项分别来自 `proxy.getDicts('baihou_space_tag')` 和 `proxy.getDicts('baihou_scene_tag')`；新增 `spaceTagsArr` / `sceneTagsArr` ref
- **规格参数动态表单**：新增 `specDefs` ref 和 `specParamsObj` ref；watch `form.categoryId`，变化时调用 `listSpecDefs(categoryId)` 加载模板、初始化 specParamsObj；模板 v-for 按 inputType 渲染 el-input / el-input-number / el-select；handleEdit 时 parse specParams 回填，submitForm 前序列化

#### 5. Banner 管理页改造（`views/baihou/banners/index.vue`）

- **投放区域字段**：原 `el-input` 纯文本 → `el-select multiple`，含”全部区域”（value=`ALL`）特殊选项和下方”全选”/”清空”快捷按钮
- **ALL 互斥逻辑**：`handleRegionChange` 中实现：选 ALL 时清除具体区域，选具体区域时清除 ALL；”全选”直接赋值 `[“ALL”]`
- **表格区域列**：原纯文本 → Tag 展示，`ALL` 使用 `type=primary`，具体区域使用 `type=info`，通过 `parseRegionTags()` 函数映射区域名称

#### 6. 品类管理页改造（`views/baihou/categories/index.vue`）

- 操作列新增”规格模板”按钮（`v-hasPermi=”['baihou:category:edit']”`）
- 新增规格模板主对话框：列出该品类的规格定义列表（specLabel/specKey/inputType/unit/isRequired/sortOrder），支持新增、编辑、删除
- 新增规格模板编辑内嵌子对话框：表单字段含显示名称/字段Key/输入类型（文本/数字/下拉选择）/选项（select类型条件渲染）/单位/必填/排序
- 规格模板状态与品类列表解耦，维护独立的 `specDefList` / `specDefForm` / `specDefOpen` 状态

### Task 1：Service 层全量补 @Transactional（P0）

**问题：** 11 个 Service impl 无任何 `@Transactional` 注解，批量操作若中途异常会导致部分记录已改、部分未改。

**修改：** 11 个 impl 文件各加 `import` + `@Transactional(rollbackFor = Exception.class)`：

`BaihouProductServiceImpl` / `BaihouBannerServiceImpl` / `BaihouCategoryServiceImpl` / `BaihouCategorySpecDefServiceImpl` / `BaihouRegionServiceImpl` / `BaihouDesignerServiceImpl` / `BaihouDesignerGroupServiceImpl` / `BaihouLeadServiceImpl` / `BaihouAppointmentServiceImpl` / `BaihouOrderServiceImpl` / `BaihouWxUserServiceImpl`

**测试：** 新建 `BaihouTransactionalAnnotationTest.java`（11 个反射断言，验证注解存在且 `rollbackFor = Exception.class`）

**提交：**
- `e9dec7c1` feat(backend): add @Transactional to all service impls for data consistency
- `36a05af5` test: strengthen @Transactional assertions to verify rollbackFor = Exception.class
- `98697420` style(test): replace wildcard import with explicit imports in TransactionalAnnotationTest

### Task 2：pageSize 后端上限 MAX=100（P0）

**问题：** pageSize 完全由客户端控制，`pageSize=1000000` 可触发全表扫描导致 OOM。

**修改：** `BaihouProduct.setPageSize()` 和 `BaihouLead.setPageSize()` 截断逻辑；常量 `MAX_PAGE_SIZE = 100`。

> BaseEntity 无 pageSize setter，故在各 domain 中分别处理；BaihouLead 原先缺少 pageSize 字段，一并新增。

**测试：** 新建 `BaihouPageSizeLimitTest.java`（4 个用例：大值截断、小值保留、null 保留、Lead 截断）

**提交：** `cfa3495d` fix(backend): cap pageSize at 100 to prevent DoS via large queries

---

### Task 3：Banner 后端时间交叉校验（P1）

**问题：** 前端已有 endTime > startTime 校验，后端未做服务端保护，可绕过前端直接 POST 无效数据。

**修改：** `BaihouBannerServiceImpl` 新增 `validateTime(BaihouBanner)` 私有方法，在 `insertBanner` 和 `updateBanner` 中各调用一次。

**测试：** `BaihouBannerServiceImplTest.java` 追加 3 个用例（end<start 抛异常、end==start 抛异常、null 时间允许通过）

**提交：** `ac801b01` fix(backend): validate banner endTime > startTime in service layer

---

### Task 4：小程序 request.js 错误对象格式统一（P1）

**问题：** 业务错误 `reject(response)` 为裸对象，网络错误 `reject({ code: statusCode })` 字段名不统一，调用侧多处 `err.msg || '操作失败'` 防御写法。

**修改：**
- `request.js` 3 种 reject 路径统一为 `new Error()`，挂载 `.code` / `.data` 属性
- 15 处调用侧 `err.msg` → `err.message`（覆盖 pages/ 和 services/ 下全部文件）

**提交：** `92f69f31` fix(miniapp): standardize request error objects to Error class

---

### Task 5：小程序 Token 401 自动静默重新登录（P1）

**问题：** JWT 过期返回 401 时只展示 toast，用户被强制踢出，长会话体验差。

**修改：**
- `services/auth.js` 新增 `silentLogin()` 函数：调用 `login()` → `userStore.setAuth()`，导出
- `request.js` success 回调最前插入 401 拦截：调 `silentLogin()` → 重放原请求；silentLogin 失败则 `userStore.logout()` + 跳转 `/pages/auth/index`

**防循环机制：** silentLogin 失败时 catch 捕获并终止链路，不会递归重放。

**提交：** `e0005b1e` feat(miniapp): auto silent re-login on 401 and retry original request

---

### Task 6：Controller 层加 @Validated + Bean Validation（P2）

**问题：** Controller 接收 JSON 无输入验证，`BaihouProduct.status` 可传任意字符串破坏状态机，`BaihouDesigner.phone` 无格式校验。

**修改：**
- `BaihouProduct.status` 加 `@Pattern(regexp = "^(draft|on_shelf|off_shelf|archived)$")`
- `BaihouDesigner.phone` 加 `@Pattern(regexp = "^1\\d{10}$")`（允许 null，不加 @NotNull）
- `BaihouProductController` / `BaihouDesignerController` 的 `add` / `edit` 方法加 `@Validated`

> 注意：项目使用 Spring Boot 2.x，验证 API 包名为 `javax.validation`（非 `jakarta.validation`）。

**测试：** 新建 `BaihouBeanValidationTest.java`（5 个用例）

**提交：** `54106bab` fix(backend): add @Validated and Bean Validation to Product/Designer controllers

---

### Task 7：订单/预约区域筛选改为下拉选择（P2）

**问题：** orders 和 appointments 页面区域筛选为 `el-input`（需手填 ID），与其他页面 `el-select` 体验不一致。

3. `onMounted` 加载区域选项
4. `el-input` 替换为 `el-select`

**提交：** `118db885` fix(admin-ui): change region filter from text input to dropdown in orders/appointments
