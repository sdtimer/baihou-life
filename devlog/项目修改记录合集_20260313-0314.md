# 柏厚生活项目开发修改记录合集

本文档汇总了柏厚生活（Baihou Life）一期项目在 2026-03-13 至 2026-03-14 期间的所有开发、修复与迭代记录。

---

## 2026-03-14 修改记录 (今日)

### 1. 核心功能修复与增强
- **小程序登录链路对齐测试态实现**
  - 前端接入 `wx.login()`，后端按测试映射返回 token、过期时间和用户信息。
  - 结果：登录态、角色态、区域态可统一缓存并用于后续页面鉴权。
- **请求层补业务状态判断**
  - 原前端只按 HTTP 2xx 视为成功，现已能正确识别 RuoYi `code != 200` 的业务失败。
  - 结果：请求封装已按业务 `code` 正确 reject，页面能收到真实错误态。
- **商品列表补齐筛选、排序、分页追加**
  - 按一期文档补齐 `keyword/category/space/scene/sort_by/pageNum/pageSize`。
  - 结果：支持滚动分页、筛选重置、排序切换、空态和错误态。
- **商品详情权限态与区域态收口**
  - 统一游客、客户、设计师三种价格和动作入口；补跨区域不可售提示。
  - 结果：详情页按角色展示价格和素材入口，游客不再直接走成交入口。
- **设计师素材下载链路补齐**
  - 流程改为“换 token 后下载，失败再复制链接”的测试态流程。
  - 结果：补全了设计师权限拦截、素材空态及下载失败兜底。
- **留资与预约表单校验完善**
  - 补全手机号、姓名、服务类型、日期校验，增加提交中状态及结果反馈。
- **新增预约详情能力**
  - 后端新增预约详情接口，前端新增预约详情页并接入列表跳转。
- **订单与测试态模拟支付闭环**
  - 拆出预支付和支付成功回写接口，接入模拟支付成功/取消/失败路径。
  - 结果：订单创建、待支付、已支付、详情继续支付链路已全线可演示。

### 2. 关键 Bug 修复 (Review 汇总)
- 修复登录后区域不同步问题。
- 修复商品详情不随城市切换刷新的问题。
- 修复支付取消后，待支付订单丢失入口的问题。

### 3. 文档口径更新
- 将一期文档和小程序 README 更新为“测试态登录 + 测试态模拟支付”，确保实现与文档一致。

## 2026-03-13 修改记录 (昨天)

### 1. 小程序前端重构与修复
- **服务层 (Services) 数据规范化重构**
  - **`normalizeProduct()`**：统一处理后端返回字段差异（驼峰/下划线互转、空值兜底）。
  - **`parseArray()` / `parseSpecs()`**：兼容标签及规格参数的多种格式（字符串/JSON/数组）。
  - **`normalizeMediaList()`**：统一场景图、规格图、素材文件的媒体列表字段。
  - **Mock 模式增强**：新增分页支持及 `region_id` 过滤。
- **工具类 (Utils) 增强**
  - **`request.js`**：新增对 RuoYi 标准业务码的支持，确保业务错误能被页面捕获。
- **页面与组件优化**
  - **首页 (`home/index.js`)**：优化 `bootstrap()` 兼容性，改进区域定位判定逻辑，补全显式分页参数。
  - **商品卡片 (`bh-product-card`)**：修复价格显示逻辑，增加空值判断及角色权限区分展示。
  - **路由配置**：在 `app.json` 中新增预约详情页路由。

### 2. 后端开发与 SQL 迭代
- **业务接口完善**
  - **商品服务**：优化 `BaihouProductServiceImpl` 返回逻辑，确保媒体文件路径拼接正确。
  - **用户鉴权**：完善 OpenID 与系统角色的映射，支持测试态 Token 签发。
  - **配置与留资**：提供基础配置聚合查询接口，补全留资表单提交校验。
- **数据库路径与数据修正**
  - **`11_fix_media_urls.sql`**：批量修正数据库中商品图片、素材的相对路径。
  - **测试数据补齐**：通过 `09_test_data.sql` 等脚本补齐了 20+ 款商品及其关联媒体数据。
